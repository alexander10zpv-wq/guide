# Урок 17. Какой тип выбрать в разных ситуациях

Главное правило: **выбирай тип не по тому, что он может хранить, а по смыслу данных и операциям, которые ты будешь с ними делать**.

## Шпаргалка по типичным задачам

| Задача                              | Лучший тип        |
|-------------------------------------|-------------------|
| Целые числа                         | `int`             |
| Дробные приблизительные числа       | `float`           |
| Деньги                              | `Decimal`         |
| Текст                               | `str`             |
| Список элементов                    | `list`            |
| Фиксированный набор                 | `tuple`           |
| Уникальные элементы                 | `set`             |
| Ключ-значение                       | `dict`            |
| JSON с известной структурой         | `TypedDict`       |
| Бизнес-объект                       | `dataclass`       |
| Дата и время                        | `datetime`        |
| Статусы                             | `Enum`            |
| Очередь                             | `deque`           |
| Подсчёт повторов                    | `Counter`         |
| Бинарные данные                     | `bytes`           |
| Изменяемые бинарные данные          | `bytearray`       |
| Отсутствие значения                 | `None`            |

## Парные решения

### `list` или `tuple`

- `list`: данные могут меняться, нужно сортировать/добавлять/удалять.
- `tuple`: данные фиксированы (координаты, RGB, возврат пары значений).

### `list` или `set`

- `list`: важен порядок, нужны индексы, могут быть дубли.
- `set`: уникальные значения, быстрая проверка `in`, порядок не важен.

### `dict` или `dataclass`

- `dict`: структура динамическая, JSON, ответ API.
- `dataclass`: структура известна, это бизнес-объект, важна читаемость.

### `float` или `Decimal`

- `float`: физика, графики, координаты — скорость важнее абсолютной точности.
- `Decimal`: деньги, бухгалтерия, точное округление.

### `str` или `bytes`

- `str`: текст, пользовательский ввод, JSON, логи.
- `bytes`: файл в бинарном режиме, сеть, изображения, шифрование.

### `Enum` или `str`

- `str`: значение свободное, текст приходит от пользователя.
- `Enum`: фиксированный список (статусы, роли, режимы).

### `dict` или `TypedDict`

- `dict`: структура свободная.
- `TypedDict`: структура известна, нужна типизация.

## Итог: «по умолчанию»

В большинстве случаев:

- `int` для целых чисел,
- `float` для приблизительных дробных расчётов,
- `Decimal` для денег,
- `str` для текста,
- `list` для обычных списков,
- `tuple` для фиксированных структур,
- `dict` для ключ-значение,
- `set` для уникальности,
- `dataclass` для бизнес-моделей,
- `Enum` для фиксированных статусов,
- `datetime` для дат и времени,
- `None` для отсутствия значения.
