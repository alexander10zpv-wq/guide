# Задача урока 17 — выбор типа

Открой [task.py](task.py) и заполни **строковые** значения, отмеченные `//TODO`. Этот урок проверяет, что ты понимаешь, какой тип взять для конкретной задачи.

## Что нужно сделать

В переменных вида `answer_*` укажи **строкой** название подходящего типа. Используй маленькие буквы и точные имена:

- `"int"`, `"float"`, `"Decimal"`, `"str"`, `"bool"`,
- `"list"`, `"tuple"`, `"set"`, `"dict"`,
- `"dataclass"`, `"Enum"`, `"datetime"`, `"deque"`, `"Counter"`, `"bytes"`, `"None"`, `"TypedDict"`.

Задачи:

1. `answer_money` — хранение цены, чтобы не было ошибок округления.
2. `answer_user_id` — уникальный целочисленный идентификатор пользователя.
3. `answer_coordinates` — фиксированная пара координат `(x, y)`, не меняется.
4. `answer_unique_tags` — набор уникальных тегов поста.
5. `answer_order_status` — фиксированный список статусов заказа: new/paid/shipped/cancelled.
6. `answer_business_user` — структурированный объект пользователя в backend-логике.
7. `answer_word_frequency` — подсчёт количества повторений каждого слова в тексте.
8. `answer_task_queue` — очередь задач, из которой берём «по очереди» (FIFO).
9. `answer_event_timestamp` — момент времени какого-то события.
10. `answer_missing_value` — «значение отсутствует / не найдено».

## Как запускать и проверять

1. Открой терминал (`Ctrl + ~`).
2. `cd lesson_17_choosing_types`
3. `python check.py`
