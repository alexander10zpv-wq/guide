# Урок 7. Множества `set` и `frozenset`

## `set` — множество уникальных элементов

```python
numbers = {1, 2, 3, 3, 3}
print(numbers)  # {1, 2, 3}  (дубли убрались)
```

Создание пустого множества — **только** через `set()`. `{}` — это пустой словарь!

```python
empty_set = set()
empty_dict = {}
```

## Когда использовать `set`

- Убрать дубликаты.
- Быстро проверить наличие элемента (`x in s`).
- Операции пересечения / объединения / разности.
- Уникальные ID, теги.

```python
ids = [1, 2, 2, 3, 4, 4]
unique = set(ids)   # {1, 2, 3, 4}
```

## Операции

```python
s = {1, 2, 3}

s.add(4)             # добавить
s.remove(2)          # удалить (ошибка, если нет)
s.discard(999)       # удалить (без ошибки, если нет)
s.pop()              # удалить случайный
len(s)               # размер
3 in s               # True
```

## Алгебра множеств

```python
a = {1, 2, 3}
b = {3, 4, 5}

a | b   # объединение            {1, 2, 3, 4, 5}
a & b   # пересечение            {3}
a - b   # разность               {1, 2}
a ^ b   # симметричная разность  {1, 2, 4, 5}

a.issubset({1, 2, 3, 4})   # True
a.issuperset({1, 2})       # True
```

## Почему `set` быстрее `list` для `in`

- `x in list` проверяет элементы по очереди → медленно на больших данных.
- `x in set` использует хеш → почти мгновенно.

## `frozenset` — неизменяемое множество

```python
permissions = frozenset(["read", "write"])
```

Когда использовать:

- Когда множество не должно меняться.
- Когда множество нужно как ключ словаря (обычный `set` ключом быть не может).

## Что НЕ хранится в `set`

- Изменяемые объекты (списки, словари) — не hashable, ошибка.
- Порядок не гарантирован (в отличие от `dict`).
